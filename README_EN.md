A bridge service that connects the Feishu (Lark) messaging platform to the [Kiro CLI](https://kiro.dev) AI agent framework. Users interact with Kiro's capabilities—such as infrastructure analysis, report generation, and automated troubleshooting—directly through a Feishu chat interface.

**No public IP required. No port forwarding. No nginx reverse proxy.**

## ✨ Core Features

| Feature | Description |
|---------|-------------|
| 🔌 **WebSocket Long-Connection** | Outbound connection, natively穿透s NAT/firewall, zero network config |
| 🧠 **Dual-Layer Memory** | Semantic Memory (user preferences) + Episodic Memory (system events), SQLite-based |
| ⚡ **Hybrid Execution Engine** | Sync 120s → auto background async with progress heartbeats |
| 🗂️ **Session Routing** | 30min auto-resume, supports `/new` `/resume` `/sessions` |
| 📎 **Auto Resource Upload** | Detects image/file paths in Kiro output, uploads to Feishu automatically |
| ⏰ **Scheduled Tasks** | Natural language cron jobs, `/schedule` command management |
| 📝 **Event Ingestion** | `/event` manual entry + Webhook external system push |

## 🚀 Quick Start

### 1. Feishu Open Platform Setup

1. Visit https://open.feishu.cn/app and log in
2. Create a custom app, record **App ID** and **App Secret**
3. Add "Bot" capability
4. Event Subscription → Choose **"Use Long Connection"** → Add `im.message.receive_v1`
5. Permissions → Enable `im:message`, `im:message:send_as_bot`, `im:resource`
6. Version Management → Submit for review → Publish

> Full permissions in `feishu-auth.json`. Minimum: `im:message` + `im:message:send_as_bot`.

### 2. Configure Service

```bash
cd /home/ubuntu/feishu-kiro-bot
cp .env.example .env
# Edit .env, fill in FEISHU_APP_ID and FEISHU_APP_SECRET
```

**Optional Config:**

| Variable | Description | Default |
|----------|-------------|---------|
| `KIRO_TIMEOUT` | Kiro CLI sync timeout (s) | `120` |
| `KIRO_ASYNC_TIMEOUT` | Async task max wait (s) | `1800` |
| `KIRO_AGENT` | Kiro agent name | empty |
| `ENABLE_MEMORY` | Enable memory feature | `false` |

### 3. Start Service

```bash
# Foreground debug
./start.sh

# systemd background (production)
sudo cp feishu-kiro-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable feishu-kiro-bot
sudo systemctl start feishu-kiro-bot
```

### 4. View Logs

```bash
sudo journalctl -u feishu-kiro-bot -f
```

## 🧠 Memory System

Memory is disabled by default. Set `ENABLE_MEMORY=true` in `.env` to enable.

**Zero additional dependencies**, all powered by Python built-in `sqlite3`.

### Dual-Layer Memory

| Type | Content | Retrieval | Prompt Injection |
|------|---------|-----------|-----------------|
| **Semantic** | User preferences, facts, decisions | Keyword overlap scoring | Prefix injection, affects Agent behavior |
| **Episodic** | System changes, deploys, alerts, incidents | Time + Entity + Type filter | Appended after user prompt, marked "for reference only" |

### Event Ingestion

**Method 1: Manual via Feishu**

```
/event type=系统变更 entities=test1,MySQL title="test1 index optimization" description="added composite index"
```

**Method 2: External Webhook Push**

```json
{
  "id": "jenkins-12345",
  "event_type": "应用发版",
  "title": "Order Service v2.3.1 deployed",
  "description": "fixed payment callback timeout",
  "entities": ["order-service"],
  "source": "jenkins",
  "user_id": "ou_xxx"
}
```

### Memory Management Commands

| Command | Function |
|---------|----------|
| `/memory status` | Show memory state and semantic memory count |
| `/memory on` | Enable memory |
| `/memory off` | Disable memory |
| `/memory clear` | Clear semantic memories |
| `/memory events` | List recent 30-day events |
| `/memory events clear` | Clear all event records |

## 💬 Multi-Turn Conversations

### Automatic Session Continuation

By default, the bot automatically continues the context of the same topic. If you send another message **within 30 minutes**, it will auto-resume the same session, and Kiro CLI will carry the full conversation history for reasoning.

### Explicit Session Management

When you need to switch topics, use these commands:

| Command | Description |
|---------|-------------|
| `/new` | Force start a new session; next message will not be affected by historical context |
| `/sessions` | View the 10 most recent historical sessions |
| `/resume <id>` | Resume a specific historical session and continue the previous conversation |

> 💡 **Tip**: If the bot's reply drifts off-topic (e.g., resumed an old session), send `/new` to reset.

## ⏰ Scheduled Tasks

Configure periodic tasks via natural language. The bot will automatically execute Kiro commands at the specified time and push results to you.

**Usage examples:**
```
/schedule check AWS cost every day at 9am
/schedule backup database every Monday at 2am
/schedule check EC2 instance status every 30 minutes
```

**Management commands:**
```
/schedule list      # List all scheduled tasks
/schedule delete 1  # Delete task #1
/schedule help      # Show help
```

## 📎 Image & File Delivery

The bot supports automatic detection of file paths in Kiro output and uploads them to Feishu:

- **Images**: `.png` `.jpg` `.jpeg` `.gif` `.bmp` `.webp` → sent as image messages
- **Files**: `.pdf` `.doc` `.docx` `.xls` `.xlsx` `.ppt` `.pptx` `.csv` `.txt` `.zip` `.mp4` → sent as file messages

**Workflow:**
1. You send a request to the bot (e.g., "generate a CPU trend chart")
2. Kiro processes and generates a file, outputting an absolute path (e.g., `/tmp/report/cpu.png`)
3. The bot automatically detects the existing file path
4. Uploads to Feishu and replies as an image/file message

> **Note**: Requires `im:resource` permission in the Feishu Open Platform (for uploading images/files).
